# Omfg 0.2.0

This release introduces a user-oriented command interface. A complete setup remains available as
`omfg` or `omfg setup`; focused operations use `apps`, `git`, `github`, `ssh`, and `codex`, while
`omfg status` performs read-only readiness checks.

Application categories are positional under `omfg apps`, and dependencies, package sources,
Flatpak, Flathub, and shell prerequisites are resolved automatically.

Migration examples:

- `omfg --apps` becomes `omfg apps`;
- `omfg --app browser` becomes `omfg apps browser`;
- `omfg --github` becomes `omfg github`;
- `omfg --check` becomes `omfg status`.

Package selection, SSH safeguards, Codex isolation, bootstrap integrity, and the stage-based setup
output are unchanged.
